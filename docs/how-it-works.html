仕組み

3層アーキテクチャ、セキュリティ設計、初回オンボーディングの流れ。

ARCHITECTURE

3層アーキテクチャ

Skill（固定・全企業共通）+ Knowledge Store（可変・企業固有）+ MCPツール（API操作）の3層で構成。

🎯 Claude Skill（判断規範）

MA担当者としての行動規範。「Knowledge Storeを毎回読め」「記載がないことは推測するな」「施策後は必ず記録しろ」。全企業共通。書き換え不要。

🧠 Knowledge Store（企業の記憶）

設計判断・命名規則・手順書・禁止事項・過去施策等の12カテゴリ。HubSpot内のCRMノートに保存。追加インフラ不要。初回は自動分析+質問で構築、以降は施策実行のたびに自動成長。

🔧 128 MCPツール（操作能力）

CRM・フォーム・リスト・メール・WF・キャンペーン・イベント等、HubSpot APIを完全カバー。Skillの指示とKnowledgeの知識に基づいて動く「手足」。

SECURITY

🔒 セキュリティ

公開エンドポイントにHubSpotトークンを送る安全性について

✅ HTTPS（TLS暗号化）で保護

全通信はHTTPS上で暗号化されます。トークンが平文で流れることはありません。

✅ ステートレス — 保存もログもしない

トークンはリクエスト処理中にのみ使用され、終了と同時に破棄されます。データベースもログ出力もありません。

✅ ソースコード全公開

処理の全てを GitHub で確認できます。

🏢 不安な場合は自分でデプロイ

Vercel/Cloudflareにデプロイして api_key モードで運用すれば、トークンをサーバー環境変数に固定。外部に一切送信しません。

FIRST SESSION

初回オンボーディング

セットアップ完了後、最初の会話でAIが自動的にあなたのHubSpotを学習します。

1AIが自動分析を開始

最初の指示で、AIがKnowledge Storeが空であることを検知。hubspot_knowledge_build を自動実行し、あなたのHubSpotの全設定を一括スキャンします。

212カテゴリの下書きを自動生成

スキャン結果から下書きを自動生成。さらにAIが自己検証ループを実行し、自力で答えを出せるものは全て解決します。

3AIからあなたへ質問

本当に自分では判断できないことだけが仮説付きYes/Noで質問されます。例：

  • 「パイプラインが1本のみ。統合運用している認識で合っていますか？」
  • 「過去メールの件名が【レヴォル】で始まるルール。踏襲してよろしいですか？」

4あなたが回答 → Knowledge完成

回答をもとにKnowledge Storeを補完。口頭で伝えるだけでAIが整形して保存します。

5MA担当者として稼働開始

以降はAIが毎回Knowledge Storeを読み込み、あなたの会社のやり方に従って施策を提案・実行。使うほど賢くなります。
