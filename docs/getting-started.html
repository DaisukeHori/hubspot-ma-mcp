THREE OPTIONS

公開サーバーにそのまま接続するか、自分専用サーバーをデプロイするか選べます。機能・ツール数は全て同一です。

3ステップで始める

💡 まずは試したい方へ: HubSpotテストアカウントを作成すれば、本番データに影響なく全機能を試せます。

① HubSpot Private Appを作成

Settings → Integrations → Private Apps → Create。必要なスコープを付与してAccess Tokenをコピー。スコープ一覧 →

② MCPサーバーに接続

上の3つの始め方から方式を選び、クライアント別の設定をコピー。

③ Claude Skillをインストール（推奨）

hubspot-ma-operator.zip をダウンロード → Claude.ai → Settings → Customize → Skills → Upload。

AUTH MODES

認証モード

環境変数 AUTH_MODE で認証方式を切り替え。

① hubspot_token

HubSpot トークン直接渡し

各ユーザーが Authorization: Bearer <HubSpot PAT> で自分のトークンを渡す。公開サーバー・個人利用向け。

② api_key

APIキーでゲート

MCP_API_KEY でアクセス制限。HubSpot トークンは環境変数に固定。デプロイした組織だけが使えるセキュアな構成。

HubSpot Private App トークンの発行方法

  1. HubSpot にログイン → ⚙️ 設定
  2. Integrations → Private Apps → Create a private app
  3. 「Scopes」タブで下記の権限にチェック
  4. 「Create app」→ Access Token（pat-na1-xxxx...）をコピー

IMPORTANT NOTICES

⚠️ 利用上の注意

🤖 Knowledge Storeの自動作成物

初回セットアップでコンタクト1件（mcp-knowledge@system.internal）とCRMノート最大10件が自動作成されます。削除するとKnowledge Storeが失われます。

📧 メール送信は取り消せない

marketing_email_publish は配信リスト全員に送信され、取り消し不可です。

⚡ ワークフロー有効化の影響

有効化すると条件合致する全レコードに即座にアクション実行の可能性があります。

🏢 Enterprise必須機能

marketing_email_publish、single_send_email、custom_event_* は Marketing Hub Enterprise が必要です。

FAQ

よくある質問

👥 チームで使う場合

同じHubSpotトークン → 同じKnowledge Storeを共有。異なるアカウント → 完全分離。

❓ Freeプランでも使える？

CRM操作・フォーム・リスト・Knowledge Store等の大半は利用可能。メール送信・カスタムイベントはEnterprise必須。

❓ Skillなしでも動く？

はい。128ツールのAPIラッパーとして機能します。Skillを追加するとMA担当者の判断力が加わります。
