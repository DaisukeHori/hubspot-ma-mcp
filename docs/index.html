ARCHITECTURE

3層アーキテクチャ

Skill（固定・全企業共通）+ Knowledge Store（可変・企業固有）+ MCPツール（API操作）の3層で構成。

🎯 Claude Skill（判断規範）

MA担当者としての行動規範。「Knowledge Storeを毎回読め」「記載がないことは推測するな」「施策後は必ず記録しろ」。全企業共通。書き換え不要。

🧠 Knowledge Store（企業の記憶）

設計判断・命名規則・手順書・禁止事項・過去施策等の12カテゴリ。HubSpot内のCRMノートに保存。追加インフラ不要。初回は自動分析+質問で構築、以降は施策実行のたびに自動成長。

🔧 128 MCPツール（操作能力）

CRM・フォーム・リスト・メール・WF・キャンペーン・イベント等、HubSpot APIを完全カバー。Skillの指示とKnowledgeの知識に基づいて動く「手足」。

SECURITY

🔒 セキュリティ

公開エンドポイントにHubSpotトークンを送る安全性について

✅ HTTPS（TLS暗号化）で保護

全通信はHTTPS上で暗号化されます。トークンが平文で流れることはありません。

✅ ステートレス — 保存もログもしない

トークンはリクエスト処理中にのみ使用され、終了と同時に破棄されます。データベースもログ出力もありません。

✅ ソースコード全公開

処理の全てを GitHub で確認できます。

🏢 不安な場合は自分でデプロイ

Vercel/Cloudflareにデプロイして api_key モードで運用すれば、トークンをサーバー環境変数に固定。外部に一切送信しません。

AUTH MODES

認証モード

環境変数 AUTH_MODE で認証方式を切り替え。公開サーバーと組織専用サーバーに対応。

① hubspot_token

HubSpot トークン直接渡し

各ユーザーが Authorization: Bearer <HubSpot PAT> で自分のトークンを渡す。MCPサーバー自体への認証はなし。公開サーバー・個人利用向け。

② api_key

APIキーでゲート

MCP_API_KEY でサーバーアクセスを制限。HubSpot トークンは環境変数に固定。デプロイした組織だけが使えるセキュアな構成。

HubSpot Private App トークンの発行方法

  1. HubSpot にログイン → 右上の ⚙️ 設定アイコン
  2. 左メニュー「Integrations」→「Private Apps」
  3. 「Create a private app」をクリック
  4. Basic Info: アプリ名（例: MCP Server）を入力
  5. 「Scopes」タブで下記の権限にチェック
  6. 「Create app」→ 表示される Access Token（pat-na1-xxxx...）をコピー

必要なスコープ（権限）

スコープ対象ツール必須

※ CMS スコープは CMS Hub 契約がある場合のみ利用可能です。不要な場合はチェック不要です。

IMPORTANT NOTICES

⚠️ 利用上の注意

🤖 Knowledge Storeの自動作成物

初回セットアップで、HubSpotアカウント内にコンタクト1件（mcp-knowledge@system.internal）とCRMノート最大10件が自動作成されます。これらはAIの記憶領域です。削除するとKnowledge Storeの内容が全て失われます。コンタクト一覧では @system.internal で除外してください。

📧 メール送信は取り消せない

marketing_email_publish は配信リスト全員に送信され、取り消し不可です。single_send_email は1通送信ですが、宛先にコンタクトが存在しない場合コンタクトが自動作成され、さらに自動的にマーケティングコンタクトに設定されます。

⚡ ワークフロー有効化の影響

ワークフローを有効化（isEnabled: true）すると、エンロールメント条件に合致する全レコードに対して即座にアクションが実行される可能性があります。Skillが必ず確認を取りますが、ご注意ください。

📝 form_updateは全体置換

form_update はPUT（全体置換）です。省略したフィールドは削除されます。Skillはform_getで現在の定義を取得してから更新を行います。

🏢 Enterprise必須機能

marketing_email_publish、single_send_email、custom_event_define/send は Marketing Hub Enterprise が必要です。それ以外の機能はFree/Starter/Professionalでも利用可能です。

🔄 APIレート制限

HubSpot標準のレート制限（Private App: 500,000リクエスト/日、Search: 4リクエスト/秒）が適用されます。hubspot_context_snapshot は内部で複数APIを呼び出すため、頻繁な実行は控えてください。

TEAM & FAQ

チーム利用 & よくある質問

👥 チームで使う場合

同じHubSpotトークン（同じアカウント）→ 同じKnowledge Storeを共有。全員が同じ暗黙知を参照できます。異なるHubSpotアカウント → 完全に分離された独立環境。

❓ Knowledge Storeを間違えて更新したら？

hubspot_knowledge_update の mode: "replace" で上書き修正可能。コンタクトを誤削除した場合は hubspot_knowledge_setup 再実行で復元（ただし内容は再構築が必要）。

❓ Freeプランでも使える？

CRM操作・プロパティ・パイプライン・フォーム・リスト・Knowledge Store等の大半は利用可能。メール送信（publish）・Single-Send・カスタムイベントはEnterprise必須です。

❓ Skillなしでも動く？

はい。MCPサーバーだけで128ツールのAPIラッパーとして機能します。Skillを追加すると「MA担当者」の判断力（自己検証・施策パターン・禁止事項チェック等）が加わります。
